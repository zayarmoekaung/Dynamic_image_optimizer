Image Optimization API

This API dynamically fetches images from external URLs, optimizes them based on device type (detected via User-Agent), client dimensions, and Device Pixel Ratio (DPR), and serves them in the preferred format. It handles special cases like long comic strips to preserve quality and caches results for performance.

Key Features

  • Device-Aware Optimization: Resizes images for mobile (640x480), tablet (1024x768), or desktop (1920x1080), with DPR scaling for high-res screens.
  • Comic Strip Support: For tall images (aspect ratio > 3), uses width-driven resizing with a 8000px height cap and higher quality (90%).
  • Format Negotiation: Supports WebP (default), JPEG, PNG, BMP, TIFF, and GIF, based on the Accept header. Falls back to WebP for unsupported or unspecified formats.
  • Health Check: Verify service status.

Endpoints

GET /optimize
Optimize an image. Params: url (required), width, height, dpr.
Headers: User-Agent (device detection), Accept (format: image/webp, image/jpeg, image/png, image/bmp, image/tiff, image/gif; defaults to WebP).
Example: /optimize?url=https://httpbin.org/image/png (returns WebP by default)
GET /health
Returns service health status.
Example: /health
GET /api/swagger
